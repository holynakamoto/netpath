## Conventions

- [01KWCKPP28DBHPG92PJ2YN9QR1] **- **Use `typer** -- - **Use `typer.Exit(code)` for exit codes, never `sys.exit()`** — Calling `sys.exit()` directly bypasses Typer's cleanup. The correct mechanism in any Typer command is `raise typer.Exit(code=N)`. Exit code convention for netpath: 0=ok, 1=warning, 2=critical (worst severity across all probes in a run). [source: task-ways-to-improve-netpath-01kwc564vewr, importance: 0.7]. [source: task-ways-to-improve-netpath-01kwc564vewr, importance: 0.7]
- [01KWDAP1069F626ZSJFXDPRCMT] **- **PMTU and TCP/TLS latency probes run unconditionally — gating on `skip_t...** -- - **PMTU and TCP/TLS latency probes run unconditionally — gating on `skip_throughput` would suppress them incorrectly** — PMTU black-hole detection and TCP/TLS application latency are path properties, not throughput measurements. `--no-throughput` (or country-mode's implicit `skip_throughput=True`) skips iperf3/speedtest, but should not suppress these probes. If gated behind `skip_throughput=False`, country-mode sweeps would lose all PMTU and latency signals. Rule: only skip probes under `skip_throughput` when the probe is a bandwidth/throughput measurement. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.65]. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.6]
- [01KWDANWJB04EA25FXS31TM1P2] **- **`classify_hop()` API: returns `"ixp"` or `"transit"` only — `"dest"` is...** -- - **`classify_hop()` API: returns `"ixp"` or `"transit"` only — `"dest"` is determined by the caller** — `ixp.classify_hop(ip: str)` only receives an IP address and has no context to determine which ASN is the "destination." The "dest" label requires `target_asn`, which `display._build_hub_table()` already has. Folding destination detection into `ixp.py` would require a second parameter and couple the IXP module to path context. Pattern: ixp module returns `"ixp" | "transit"`; display checks `hub["ASN"] == target_asn` independently to decide "dest" vs calling `classify_hop()`. ## Patterns

- [01KWDANDKPYBXAVAWBH73J23KY] **- **ICMP rate-limit false positive: fix belongs in `diagnosis** -- - **ICMP rate-limit false positive: fix belongs in `diagnosis.py` forward-scan, not `mtr.py`** — When mid-path hop loss must be re-classified based on downstream context, the logic belongs in `diagnosis.py` (the pure-function module) rather than adding a per-hop flag in `mtr.py`. A forward-scan after finding `Loss% > 1.0` at hop i checks whether all subsequent responsive hubs are clean; if yes, suppress the alarm and emit an informational signal instead. Coupling transport-layer flag into the hub dict would mix concerns across module boundaries. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.7]. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.7]
- [01KWDAPBHDAC49H1YCS37TAK25] **- **Single-purpose measurement module convention: each new measurement concer...** -- - **Single-purpose measurement module convention: each new measurement concern gets its own file** — Project follows the pattern of iperf.py, speedtest.py, rum.py each owning one measurement concern. New measurement modules introduced in Phase 3 follow the same pattern: `pmtu.py` (ICMP MTU probing), `latency.py` (TCP connect + TLS handshake), `ixp.py` (hop classification). Benefits: independently testable with no subprocess mocking required (latency.py uses stdlib only), importable in isolation, consistent with existing codebase structure. Avoid inlining new measurement logic in cli.py. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.65]. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.6]
- [01KWDANJXWPA4XVAV51TC0AYWY] **- **In-process module-level cache for external API data (PeeringDB IXP prefix...** -- - **In-process module-level cache for external API data (PeeringDB IXP prefixes)** — When an external API provides reference data used in tight loops (e.g., country sweeps calling `classify_hop()` per hop), cache the result in a module-level list/dict populated on first call. Disk caching requires cache invalidation + writable path; in-process caching is sufficient because netpath is a CLI tool that exits after each run. Pattern: `_IXP_PREFIXES: list = []` module-level var, populated by a lazy `_load_ixp_prefixes()` guarded by `if not _IXP_PREFIXES`. Fail silently on fetch error, return a safe default. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.65]. [source: task-fixing-incomplete-paths-richer-path-01kwcya418n5, importance: 0.6]
- [01KWCKPTN09VEWAVZ9R4GZGR63] **- **netpath test strategy: target pure functions and subprocess-mockable modu...** -- - **netpath test strategy: target pure functions and subprocess-mockable modules** — Pure functions with no subprocess, network, or filesystem dependencies are the primary test targets: `diagnosis.diagnose()`, `mtr._parse_traceroute_output()`, `mtr._all_stars()`, `mtr._compare_as_paths()`, `country.get_test_ip_for_asn()`, `pmtu.probe()` (mock `subprocess.run`). Testing `_run_test`, `servers.find_servers_in_asn`, or `iperf.run` would require heavy subprocess/HTTP mocking for minimal reliability gain. Module-private functions are importable directly: `from netpath.mtr import _parse_traceroute_output`. Test files: `tests/test_diagnosis.py`, `tests/test_mtr.py`, `tests/test_country.py`.
