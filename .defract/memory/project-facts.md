## Patterns

- [01KWF7SR22DVZES7BVA5PGK4HV] **- **`diagnose()` accumulates all matched signals; worst severity sets the top...** -- - **`diagnose()` accumulates all matched signals; worst severity sets the top-level verdict** — `diagnosis.py` runs all nine diagnostic checks unconditionally and appends every matched condition to a `signals` list as `{condition: str, severity: str, detail: str}` dicts. The top-level `severity` is derived via `max(signals, key=lambda s: _SEVERITY_ORDER[s['severity']])`. `verdict_panel()` renders each signal's `detail` field. The function short-circuits to "Healthy" (severity "ok", empty signals) only when no conditions match. The outer `except Exception` safety net remains; `probe_errors` and `partial_results` are computed inside the try block so a failing probe still contributes its signal. Rate-limited hop signals use severity "ok" so they appear without elevating the verdict. [source: task-systems-design-review-of-netpath-01kwesg18dsj, importance: 0.7]. [source: task-systems-design-review-of-netpath-01kwesg18dsj, importance: 0.7]
- [01KWF7SG32BTB8AV3CFDWMNDBT] **- **`_measure()` uses a unified `probe_errors: dict[str, str]` for all probe ...** -- - **`_measure()` uses a unified `probe_errors: dict[str, str]` for all probe failures** — `_measure()` initialises `'probe_errors': {}` in the result dict at the top; every probe that fails populates it with a short reason string keyed by probe name (`v4_trace`, `v6_trace`, `iperf3`, `speedtest`, `pmtu`, `tcp`, `tls`). The old pattern of `_trace_error`, `_iperf_error`, `_speedtest_error` as top-level `_`-prefixed keys is removed. `diagnose()` reads `probe_errors` and sets `partial_results: bool`; `verdict_panel()` renders `(partial results: probe-names)` when True. Both public and `probe_errors` keys are JSON-serializable; the `MeasurementResult` TypedDict (total=False) documents the full shape.
